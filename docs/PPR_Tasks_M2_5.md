# PPR 项目任务列表 - 阶段二点五：M2.5 (文件管理系统)

本阶段目标为实现类似 Windows 资源管理器的文件存储、目录树可视化管理、文件上传下载及对外直链访问。为了适应 AI 编程模型的上下文窗口，任务已被细化为单次 PR 级别的颗粒度。

## 后端任务 (Backend)

### [x] Task B2.5.1: 基础文件存储与公开访问接口
* **执行步骤**: 
  1. 在 `application.yml` 中增加配置文件上传与存储的物理根目录路径（如 `ppr.file.storage-path`）。
  2. 编写 `FileStorageService`，封装底层文件的读写、流转换和路径安全校验（防止路径穿越 `../` 漏洞）。
  3. 开发公开接口 `GET /api/v1/file/access/**`，接收相对路径并读取服务器物理目录下的静态文件，写入 `HttpServletResponse` 输出流，根据后缀推断并设置合适的 `Content-Type`。
* **验收标准**: 
  * 在配置的物理目录下手动放入一张图片，通过浏览器直接访问 `/api/v1/file/access/test.jpg` 能够成功预览该图片内容。

### [x] Task B2.5.2: 目录层级树管理接口
* **执行步骤**:
  1. 开发 `GET /api/v1/admin/file/dir/tree` 接口，递归扫描配置的根目录，返回目录层级树结构数据（仅包含目录节点，不含文件）。
  2. 开发 `POST /api/v1/admin/file/dir/create` 接口，接收父目录路径和新目录名，在物理系统上创建对应的文件夹。
  3. 开发 `PUT /api/v1/admin/file/dir/rename` 接口，接收原目录相对路径和新目录名，执行文件夹重命名操作。
  4. 开发 `DELETE /api/v1/admin/file/dir/delete` 接口，接收目录相对路径，删除指定目录（可根据策略决定是否允许递归删除非空目录）。
* **验收标准**: 
  * 调用树结构接口能够成功获取完整层级，并在指定路径下成功执行创建、重命名、删除目录操作，且所有操作均无法越权访问存储根目录以外的文件。

### [x] Task B2.5.3: 后台文件操作与管理接口
* **执行步骤**:
  1. 开发 `GET /api/v1/admin/file/list` 接口，接收当前目录路径参数（支持分页），返回该目录下的文件列表（包含文件名、大小、最后修改时间等属性），结果不包含子目录。
  2. 开发 `POST /api/v1/admin/file/upload` 接口，接收 `MultipartFile` 以及目标目录路径参数，将上传的文件安全保存至对应物理目录。
  3. 开发 `GET /api/v1/admin/file/download` 接口，供后台管理系统使用（强制下载行为，设置 `Content-Disposition: attachment`）。
  4. 开发 `DELETE /api/v1/admin/file/delete` 接口，删除指定的单一文件。
* **验收标准**: 
  * 能够通过 API 上传文件至指定子目录，查询文件列表接口时数据能正确展示，并能成功执行后台文件的下载和物理删除操作。

## 前端任务 (Frontend)

### [x] Task F2.5.1: 文件管理页面布局与目录树集成
* **执行步骤**:
  1. 开发 `/views/file/index.vue`，采用 CSS Grid 或 Flex 建立类似 Windows 资源管理器的左右分栏结构（左侧固定宽度，右侧自适应）。
  2. 在左侧引入 Element Plus 的 `el-tree` 组件，组件加载时请求 `/api/v1/admin/file/dir/tree` 获取并渲染后台真实的目录层级。
  3. 绑定树节点的点击事件，在选中节点时更新当前全局/局部状态（即当前访问的目录相对路径）。
* **验收标准**: 
  * 页面呈现标准的左右分栏布局，左侧能正确加载多级目录树；点击不同的目录节点，控制台能正确打印出目标目录的相对路径。

### [x] Task F2.5.2: 顶部工具栏与目录操作弹窗
* **执行步骤**:
  1. 在页面右侧上方开发顶部操作栏，左边展示当前路径的面包屑导航 (`el-breadcrumb` 或输入框)。
  2. 右边放置核心操作按钮组：“新建目录”、“重命名目录”、“删除目录”。
  3. 开发对应的弹窗表单 (`el-dialog` + `el-form`)，收集新目录名称等输入，并调用相应的后台 API (`/dir/create`, `/dir/rename`, `/dir/delete`)。
  4. 各种目录操作成功完成后，触发重新加载左侧 `el-tree` 并刷新当前选中的路径状态。
* **验收标准**: 
  * 在选定某一目录后，点击新建/重命名/删除按钮，能正确弹出对话框；完成操作后后台目录结构真实发生改变，前端页面自动同步更新树形菜单。

### [x] Task F2.5.3: 文件列表展示、上传与行级操作
* **执行步骤**:
  1. 在页面右侧下方开发基于 `el-table` 的文件展示列表，集成 `el-pagination` 实现分页加载。
  2. 监听当前选中目录的变化，触发请求 `/api/v1/admin/file/list` 并在表格中渲染对应文件数据（展示名称、大小、时间等）。
  3. 在顶部工具栏补充“文件上传”按钮，集成 `el-upload`，上传时需动态附带当前选中的目录路径作为请求参数。
  4. 在表格的“操作列”增加三个功能按钮：“下载”、“删除”、“复制直链”。
  5. “复制直链”功能：自动拼接当前前端域名/后台 API 地址与 `/api/v1/file/access/` 及其相对路径，并将完整 URL 写入系统剪贴板。
* **验收标准**: 
  * 切换左侧目录时能正确拉取对应文件列表并支持翻页；通过上传按钮可成功将文件存入当前目录；点击“复制直链”能正确获取完整 URL 并在浏览器新标签页访问到原文件。
